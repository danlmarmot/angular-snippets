Why this page? Well, I needed to pass in values to Angular.js controllers, and realized there is a lot of discussion, there isn't a practical listing of examples... hence this page.

There

1. Declare the variable in the controller's HTML tag.

A simple method for injecting a static variable when you create a controller. To use in your .js file, you can bind it to $scope and then use it. In the following case, that variable is named $scope.foods, and the expected value is "golden maple syrup".

Result: {{foods}}

ng-init method

Inject a variable directly into the $scope of the controller with ng-init. No need to use $scope or anything, just use the naked variable name.

Result: {{stupid}}

2. Use the factory!

Request a simple value from a factory with a no-parameter method

Result: {{factoryFoods}}

3. Use the factory, with a parameter!

Request a value from a factory method. Declare a value, then assign the $scope.<variable> to the result of function call. Again, the action's all in the app.js file.

Result:
  • {{factoryFoodsTaco}}
  • {{factoryFoodsSushi}}

4. Use the factory, but use values defined in Javascript outside of angular

Use the factory to access global window variables. This was the original problem I was solving--I wanted my Angular page to be driven by variables declared in index.html, and not in some Angular service or other .js file. I need to use index.html as I'm moving over web pages generated with Python Flask, and it's easy in Flask to return a single index.html file the the HTTP browser client that's rendered by the Flask template engine. Again, the action is in app.js.
Expect breakfast and dinner!

Result: {{mealNames}}
